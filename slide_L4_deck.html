Blocco 01Il problema
02 · Auto-memory
四種
Un file indice + un file per ogni cosa da ricordare.

MEMORY.md — i 4 tipi.

Cos'èLa memoria persistente di Claude Code: un indice MEMORY.md + file topic separati.
A cosa serveClaude la legge a inizio sessione e la aggiorna da solo quando impara fatti durevoli.
Come si usaSi attiva da sola. Puoi forzarla: "ricordati che…". Lui salva e indicizza.
Tipo · 01
user
Chi sei: ruolo, expertise, obiettivi. Claude adatta le risposte a te.
Tipo · 02
feedback
Le tue regole su come lavorare: "mai stimare", "sempre link cliccabili".
Tipo · 03
project
Stato dei progetti attivi: decisioni, contesto, cosa è in corso.
Tipo · 04
reference
Pointer a sistemi esterni: dove vive cosa (Drive, Notion, gruppi Telegram).

Esempi reali: feedback_caveman.md · project_leishape_meta_ads.md · reference_telegram_groups.md. Rispetti la tassonomia, non sbagli mai.

Blocco 024 tipi di memoria
03 · Graphify
地図
Dalla cartella di markdown alla galassia di nodi.

Cos'è Graphify.

Cos'èUn tool che trasforma i tuoi .md in un grafo navigabile: concetti = nodi, link = archi.
A cosa serveClaude lo interroga via MCP: "i nodi più collegati a retention?" → risposta in 2 secondi.
Come si usaIndicizzi una cartella, apri il grafo HTML in browser, e Claude lo interroga da terminale.
MEMORY.md
Elenco.
Lista piatta di fatti. Il diario: chi sei, cosa è stato deciso, dove vive cosa.
Graphify
Mappa.
Ragnatela di concetti che si parlano. La mind map dell'intero vault, interrogabile.

MEMORY.md = elenco. Graphify = mappa. Servono entrambe.

Blocco 03Elenco vs mappa
04 · Install
3
Tre comandi. Cinque minuti.

Install Graphify.

# 1. Installa il tool (via uv)
$ uv tool install graphifyy
# 2. Indicizza la cartella che vuoi mappare
$ graphify ~/Documents/IlTuoVault/02_KNOWLEDGE
# 3. Collega il grafo a Claude via MCP
$ claude mcp add graphify-vault -- python -m graphify.serve <path>/graphify-out/graph.json

Si crea una cartella graphify-out/ con il grafo HTML interattivo + i file json di stato. Da quel momento il tuo vault è un grafo.

Blocco 043 comandi · 5 min
05 · Indicizzare
索引
Una sola indicizzazione e vedi tutto il tuo cervello.

Indicizzare il vault.

Fase · 01 — scan
Scansiona i .md
Legge tutti i file, estrae le entità, costruisce i nodi, calcola gli archi. 1–3 minuti per un vault medio.
Fase · 02 — grafo
Apri il grafo HTML
La galassia di nodi nel browser. Click su un nodo = vedi i vicini. Zoom sulle comunità di concetti.
Fase · 03 — naviga
Shortest path
Il percorso più corto tra due concetti qualsiasi. Come si collegano "retention" e "bundle"? Te lo mostra.

La prima volta che lo apri ti emoziona: è il tuo cervello, visto da fuori.

質問
Smetti di cercare con grep. Chiedi al grafo.

Interrogare il grafo.

— claude — MCP graphify —

› trova il path tra "retention" e "bundle"
← retention → subscription → AOV → bundle (3 hop)

← 7 decisioni trovate, in ordine cronologico… ▌

Use case reale: decisione di pricing → tutte le decisioni passate collegate, in 5 secondi. Niente che hai mai fatto a mano.

Blocco 07Include / exclude
08 · Refresh
更新
↻
Schedula e dimentica. L'oblio è il nemico.

Refresh giornaliero.

Cos'èUn job schedulato (launchd / cron) che ogni notte ri-indicizza il vault.
A cosa serveIl grafo resta fresco da solo. Tu lo apri quando serve, è sempre aggiornato.
Come si usaGiornaliero se modifichi tanto, settimanale se poco. Mai manuale.
# launchd: ogni notte alle 03:00
$ graphify ~/Documents/IlTuoVault/02_KNOWLEDGE --refresh
# Output: delta stats
← +12 nodi · +18 archi · −2 nodi rimossi

Una volta scheduled, Graphify si aggiorna da solo. Tu lo apri quando ti serve, ed è sempre fresco.
