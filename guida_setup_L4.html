01 · La mappa

Due pezzi, un cervello

Il problema: ogni chat di Claude parte da zero. La soluzione sono due sistemi che lavorano insieme — uno ricorda, l'altro collega.

01 · ricorda
MEMORY.md
La memoria automatica di Claude Code. File di testo che Claude legge a inizio sessione e aggiorna da solo quando impara qualcosa su di te o sul progetto.
02 · collega
Graphify
Trasforma il vault in un grafo: ogni concetto è un nodo, ogni relazione un arco. Claude lo interroga via MCP e vede connessioni che tu non vedi.
Regola: MEMORY.md = elenco di cose da ricordare. Graphify = mappa di come si collegano. Servono entrambi.
02 · Memoria automatica

Verifica che la memoria sia attiva

La memoria persistente è già inclusa in Claude Code — nessuna installazione. Vive in una cartella per ogni progetto. Verifica così, dentro una sessione claude:

# Dentro Claude Code, chiedi:
› dove sta il tuo file di memoria per questo progetto?

# Risposta attesa: un percorso tipo
# ~/.claude/projects/<nome-progetto>/memory/MEMORY.md
Se Claude risponde con il percorso, la memoria è attiva. Il file MEMORY.md è l'indice: ogni voce punta a un file di dettaglio nella stessa cartella. Si crea da solo alla prima cosa che Claude decide di ricordare.
03 · I 4 tipi

I 4 tipi di memoria

Claude classifica ogni ricordo in uno di 4 tipi. Conoscerli ti serve per dettare memorie precise invece di sperare che capisca da solo.

tipo · 01
user
Chi sei, che ruolo hai, cosa sai già. Es: "preferisco risposte brevi", "il mio brand vende su Shopify".
tipo · 02
feedback
Correzioni e conferme sul metodo. Es: "mai sovrascrivere i report", "i tool sempre in inglese".
tipo · 03
project
Stato del lavoro: deadline, decisioni, contesto. Es: "lo store va live il 15", "il token scade a novembre".
tipo · 04
reference
Dove stanno le cose fuori dal progetto. Es: "i bug si tracciano su quel foglio", "il gruppo Telegram giusto è questo".
04 · Prima memoria

Forza la prima memoria

Non aspettare che succeda da solo: detta tu la prima memoria e verifica che persista tra sessioni.

1
Detta la memoriaIn sessione: ricordati che [fatto importante su di te o sul progetto]. Claude salva il file e aggiorna l'indice MEMORY.md.
2
Chiudi tuttoEsci dalla sessione (Ctrl+C o chiudi il terminale). La sessione muore, la memoria no.
3
Riapri e verificaNuova sessione claude nella stessa cartella → chiedi: cosa ti ricordi di me?. Deve citare il fatto del passo 1.
Test passato = memoria operativa. Da ora ogni cosa importante che dici a Claude può sopravvivere alla chat.
05 · Graphify

Installa Graphify — 3 comandi

Graphify si installa con uv (il gestore pacchetti Python veloce). Se non hai uv, il primo blocco lo installa.

# 0. Solo se non hai uv (una volta sola)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 1. Installa il tool
uv tool install graphifyy

# 2. Verifica
graphify --help
Esce la schermata di help = installato. Attenzione al nome: il pacchetto è graphifyy (doppia y), il comando è graphify (singola).
06 · Indicizzazione

Indicizza il vault

Punta Graphify alla cartella della conoscenza — non a tutto il vault (vedi sezione 09). Per un vault medio ci mette 1–3 minuti.

# Indicizza la cartella knowledge del tuo vault
graphify ~/Documents/IlTuoVault/02_KNOWLEDGE

# Output: si crea la cartella graphify-out/ con dentro
#   graph.json   → il grafo (nodi + archi)
#   graph.html   → la visualizzazione interattiva

# Apri il grafo nel browser
open ~/Documents/IlTuoVault/02_KNOWLEDGE/graphify-out/graph.html
La prima apertura è il momento "wow": il tuo cervello visto da fuori. Click su un nodo = vedi i vicini. Le comunità di nodi = i tuoi temi ricorrenti.
07 · MCP

Collega il grafo a Claude

Il grafo nel browser è bello, ma il valore vero è quando Claude lo interroga da solo. Si collega via MCP con un comando:

# Sostituisci il path con il TUO percorso reale
claude mcp add graphify-vault -- python -m graphify.serve ~/Documents/IlTuoVault/02_KNOWLEDGE/graphify-out/graph.json

# Verifica che il server sia registrato
claude mcp list
Da questo momento, in ogni sessione Claude ha i tool del grafo: query_graph, get_neighbors, shortest_path, god_nodes. Non devi chiamarli tu — li usa lui quando serve.
08 · Verifica

Interroga il grafo — i 3 test

Nuova sessione claude, tre domande. Se rispondono tutte, il sistema è in piedi.

TestChiediRisposta attesa
01 · god nodes"quali sono i concetti più collegati del mio grafo?"Lista dei nodi con più connessioni — i tuoi temi centrali
02 · vicini"cosa è collegato a [un tuo concetto]?"I nodi adiacenti, con il tipo di relazione
03 · percorso"come si collegano [concetto A] e [concetto B]?"Il percorso più corto tra i due, nodo per nodo
Il test 03 è quello che cambia il modo di lavorare: connessioni tra idee che non sapevi di avere — su carta non le avresti mai viste.
09 · Selezione

Cosa indicizzare e cosa no

Indicizzare tutto = grafo rumoroso e inutile. La regola: dentro la conoscenza, fuori il materiale grezzo.

Dentro il grafoFuori dal grafo
02_KNOWLEDGE — pattern, decisioni, learnings04_SESSIONS — log automatici delle sessioni
01_PROJECTS — i progetti attivi e le loro note03_REFERENCE pesante — PDF, export, dump grezzi
Brains/ — le lezioni dei tuoi esperti00_INBOX — materiale non ancora processato
Criterio in una frase: se è conoscenza scritta da te o per te → dentro. Se è materiale grezzo o log automatico → fuori.
10 · Refresh

Refresh giornaliero automatico

Il vault cresce ogni giorno; il grafo deve seguirlo. Niente lavoro manuale: chiedi a Claude di schedulare il refresh, è il suo mestiere.

# In sessione claude, detta questo:
› schedula un refresh giornaliero del mio grafo Graphify alle 03:00:
  comando "graphify ~/Documents/IlTuoVault/02_KNOWLEDGE --refresh",
  su Mac usa launchd, log su file

Claude crea il file di scheduling, lo carica e lo testa. Il --refresh è incrementale: riprocessa solo i file cambiati, pochi secondi invece di minuti.

Risultato: ogni mattina il grafo riflette il vault di ieri sera. Tu non tocchi niente.
