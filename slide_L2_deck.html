Blocco 02bIDE + motore
03 · File madre
Il file delle regole. Letto a ogni sessione.

CLAUDE.md

Cos’èUn file di testo .md che Claude legge a ogni sessione.
A cosa serveDirgli una volta sola le regole: lingua, stile, dove salvare, cosa evitare.
Come si usaCrei CLAUDE.md nella root del progetto. Lui lo applica sempre, da solo.
# CLAUDE.md

## Lingua
- Rispondi sempre in italiano.

## Stile
- Risposte brevi, niente fronzoli.
- Mai inventare dati che non hai.

## File
- Leggi un file prima di modificarlo.
- Mai sovrascrivere CSV/XLSX esistenti.

## Notifiche
- A fine task, manda un messaggio nel canale giusto.

~/.claude/CLAUDE.md = globale, vale ovunque.  ·  ./CLAUDE.md = locale, sovrascrive solo qui.

Blocco 06Ponti esterni
06 · CLI
La shell del Mac. Il backstop universale.

CLI.

Cos’èCLI = Command Line Interface. La shell del Mac (zsh) — gli stessi comandi che useresti tu manualmente. Claude scrive e lancia comandi al posto tuo.
A cosa serveBackstop universale: quando per quel tool non esiste un MCP, c'è quasi sempre un comando CLI. Decine di migliaia di tool open-source sono già "lingua nativa" per Claude.
Come si usaChiedi a parole ("comprimi questo video", "fai commit", "scarica questo YT"). Claude scrive il comando esatto, ti chiede conferma, lo esegue, ti mostra l'output.
— zsh — claude code —
$ ffmpeg -i raw.mov -crf 30 out.mp4 # 1 GB → 80 MB
$ yt-dlp <url> # scarica video competitor
$ gh pr create --title "fix bug" # pull request
$ shopify theme push # deploy tema live
$ curl -s api.com/orders | jq # JSON pulito
$ python pipeline.py ▌
Video / audio
ffmpeg · whisper · yt-dlp
Codice / deploy
git · gh · shopify · vercel
Web / API
curl · jq · wget
Dati / file
python · sed · grep · awk
Sistema / Mac
brew · open · pmset · osascript

Regola pratica: prima cerchi un MCP. Se non c'è, cerchi il tool CLI. Quasi sempre esiste — ed è già installato o si installa con brew install <nome>.

11b · Setup Mac
⌘
設定
Da zero a operativo in 5 comandi.

Setup Mac.

Cos’èL'installazione passo-passo di Claude Code sul tuo Mac, partendo da macchina pulita.
A cosa serveAvere Claude Code che gira in terminale, pronto a ricevere comandi.
Come si usaCopi-incolli i 5 comandi in ordine. 10 minuti totali, autenticazione inclusa.
# 1. Homebrew (se non ce l'hai già)
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
# 2. Node.js
$ brew install node
# 3. Claude Code
$ npm install -g @anthropic-ai/claude-code
# 4. Autenticazione (apre il browser)
$ claude # login con account Anthropic
# 5. Antigravity (opzionale, raccomandato)
$ brew install --cask antigravity

12b · Windows
⊞
窓
Funziona su Windows, ma passa da Linux.

Windows · WSL2.

Cos’èWSL2 = un Linux che gira dentro Windows. Claude Code richiede WSL2 (non Windows nativo).
A cosa serveFar girare il terminale Unix che Claude Code si aspetta. Stessa esperienza del Mac.
Come si usaInstalli WSL2 (1 comando PowerShell), poi gli stessi 5 passi del Mac dentro la shell Linux.
# In PowerShell come admin
> wsl --install
# Riavvii, apri Ubuntu, poi:
$ curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
$ sudo apt install nodejs
$ npm install -g @anthropic-ai/claude-code
$ claude

Io uso Mac e Windows non lo padroneggio. Per la guida ufficiale aggiornata → docs.claude.com/en/docs/claude-code
